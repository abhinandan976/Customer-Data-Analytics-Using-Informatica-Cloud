Data Schema

Below is the schema of how our datasets are related:

Customer Data

  • customer_id - VARCHAR(10) PRIMARY KEY
  • first_name - VARCHAR(50)
  • last_name - VARCHAR(50)
  • EVS - VARCHAR(100)
  • phone_number - VARCHAR(15)
  • created_date - DATE
  • country_id - VARCHAR(5)

2. ecommerce_purchases

  • order_id - VARCHAR(10) PRIMARY KEY
  • customer_id - VARCHAR(10), FOREIGN KEY REFERENCES customer_crm(customer_id)
  • product_id - VARCHAR(10)
  • amount - DECIMAL(10,2)
  • purchase_date - DATE
  • payment_method - VARCHAR(50)

3. support_tickets

  • ticket_id - VARCHAR(10) PRIMARY KEY
  • customer_id - VARCHAR(10), FOREIGN KEY REFERENCES customer_crm(customer_id)
  • issue_type - VARCHAR(50)
  • status - VARCHAR(20)
  • opened_date - DATE
  • closed_date - DATE

4. marketing_contacts

  • campaign_id - VARCHAR(10) PRIMARY KEY
  • customer_id - VARCHAR(10), FOREIGN KEY REFERENCES customer_crm(customer_id)
  • channel - VARCHAR(50)
  • interaction_date - DATE
  • action_taken - VARCHAR(50)

5. customer_summary_metrics

  • customer_id - VARCHAR(10) PRIMARY KEY
  • full_name - VARCHAR(100)
  • EVS - VARCHAR(100)
  • phone_number - VARCHAR(15)
  • total_purchases - INT
  • total_spent - DECIMAL(10,2)
  • last_purchase_date - DATE
  • total_support_tickets - INT
  • last_ticket_status - VARCHAR(20)
  • last_campaign_action - VARCHAR(50)
